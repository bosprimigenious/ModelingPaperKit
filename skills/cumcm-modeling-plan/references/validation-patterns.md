# Validation Patterns

- Holdout validation: split labeled data into train/test.
- Cross-validation: use when sample size is limited.
- Backtesting: train on earlier time and test on later time.
- Residual analysis: check distribution, trend, and outliers.
- Constraint check: verify optimization outputs satisfy all hard constraints.
- Conservation check: flows, totals, mass, or budget balance.
- Benchmark comparison: compare with baseline or known heuristic.
- Scenario validation: test plausible edge cases.

Always connect validation metrics to the problem objective.
