# CUMCM Citation AI Log

Use this skill for source integrity and AI-use compliance.

## Workflow

1. Separate ordinary references, data sources, software/tool references, and AI-tool records.
2. For AI use, capture tool name, version/model, provider, date, use purpose, and human verification.
3. If no AI tools were used, keep the no-AI statement after references if current rules require it.
4. If AI tools were used, ensure:
   - relevant text is marked where required
   - tool entries appear in references
   - supporting material contains the AI-use details PDF or equivalent file
5. Read `references/checklist.md` for the review fields.
6. Read `references/source-log-schema.md` for source tracking, `references/ai-use-log-schema.md` for AI records, and `references/reference-examples.md` when editing LaTeX references.
7. Read `references/examples.md` when handling real citation cleanup or AI-use compliance.

## Guardrails

- Do not invent citations.
- Do not cite a source that was not actually used.
- Use official contest instructions for current AI policy; browse when current wording matters.
