# CUMCM 2026 Rules Watch

Use this skill to keep ModelingPaperKit aligned with official 2026 CUMCM requirements.

## Workflow

1. Treat official CUMCM sources as primary. Browse when the request depends on current rules, dates, notices, or AI policy.
2. Record exact dates and source URLs when reporting any rule.
3. Compare current guidance against:
   - `templates/cumcm/main_cumcm.tex`
   - `templates/cumcm/sections/cover.tex`
   - `templates/cumcm/sections/numbering_page.tex`
   - `templates/cumcm/sections/references.tex`
   - `README.md`
   - `skills/modeling-paperkit/references/cumcm.md`
4. If changing files, keep edits narrow and explain which official requirement each edit supports.
5. Read `references/checklist.md` when performing a rule audit.
6. Read `references/official-sources.md` when deciding which sources are authoritative.
7. Read `references/examples.md` when shaping a recurring rules-monitoring task.

## Guardrails

- Do not rely on old forum posts or copied templates as authoritative.
- Do not browse public discussion of live contest problems during the active contest window unless the user explicitly asks and contest-safety rules allow it.
- Do not claim the template is submission-ready unless build and submission checks have been run or the missing checks are clearly disclosed.
