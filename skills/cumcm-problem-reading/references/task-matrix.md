# Task Matrix

Use this table shape when converting a problem statement into work:

| Subproblem | Required answer | Data needed | Candidate method | Output artifact | Validation |
| --- | --- | --- | --- | --- | --- |
| Q1 | numeric, ranking, prediction, design, or explanation | files, fields, units | baseline and likely main method | table, figure, formula, algorithm | holdout, sensitivity, consistency, expert rule |

Keep rows short. Put uncertainty in a separate notes section instead of hiding it in the method column.
