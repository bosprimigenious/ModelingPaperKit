# Section Skeletons

## Problem Restatement

- Background in the team's own words.
- Data and constraints from the problem statement.
- Numbered subproblems matching the original tasks.

## Assumptions

- Assumption statement.
- Reason or evidence.
- Later use in model or solution.

## Notation

- Symbol.
- Meaning.
- Unit.
- Scope, if symbol is reused across subproblems.

## Model And Solution

For each subproblem:

1. Objective.
2. Variables and parameters.
3. Model equations or algorithm.
4. Solving method.
5. Result and interpretation.
6. Validation or sensitivity.

## Evaluation

- Strengths tied to evidence.
- Limitations tied to assumptions or data.
- Improvement directions that are realistic inside the model family.
